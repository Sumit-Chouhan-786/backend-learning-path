Node.js, Express, MongoDB, SQL, JWT, and Image Storage Learning Path with Examples

A structured guide with examples to master full-stack development using Node.js, Express.js, MongoDB, SQL, JWT, and Image Storage.

Node.js Learning Path

  • Introduction to Node.js

    Node.js is a JavaScript runtime that lets you run JavaScript on the server-side.

    console.log('Hello from Node.js');
  • Event Loop in Node.js

    The event loop handles asynchronous tasks in Node.js.

    setTimeout(() => console.log('Delayed Log'), 2000);
  • Modules in Node.js

    Node.js uses modules for code reuse. You can use built-in modules or create your own.

    const fs = require('fs');
fs.readFile('example.txt', 'utf8', (err, data) => {
    if (err) throw err;
    console.log(data);
});
  • HTTP Module

    The HTTP module allows you to create a server that listens for incoming requests.

    const http = require('http');
const server = http.createServer((req, res) => {
    res.end('Hello World');
});
server.listen(3000);

Express.js Learning Path

  • Introduction to Express.js

    Express.js is a web application framework for Node.js, simplifying server-side development.

    const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Hello World'));
app.listen(3000);
  • Routing in Express.js

    Express provides an easy way to handle routing.

    app.get('/about', (req, res) => {
    res.send('About page');
});
  • Middleware in Express.js

    Middleware functions are used to modify the request/response objects.

    app.use((req, res, next) => {
    console.log('Request received');
    next();
});
  • Error Handling in Express.js

    Express has built-in mechanisms for error handling.

    app.use((err, req, res, next) => {
    res.status(500).send('Something went wrong!');
});

MongoDB Learning Path

  • Introduction to MongoDB

    MongoDB is a NoSQL database that stores data in flexible JSON-like format.

    const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost/myapp');
const Schema = mongoose.Schema;
const UserSchema = new Schema({ name: String });
const User = mongoose.model('User', UserSchema);
  • CRUD Operations in MongoDB

    Performing CRUD operations using MongoDB and Mongoose.

    // Create
const user = new User({ name: 'John' });
user.save();
// Read
User.find({ name: 'John' });
// Update
User.updateOne({ name: 'John' }, { $set: { name: 'Doe' } });
// Delete
User.deleteOne({ name: 'John' });

JWT (JSON Web Token) Learning Path

  • Introduction to JWT

    JWT is an open standard for securely transmitting information between parties as a JSON object. It is used for authentication and information exchange in web applications.

    const jwt = require('jsonwebtoken');

const token = jwt.sign({ userId: '123' }, 'secretkey', { expiresIn: '1h' });
console.log(token);
  • JWT Authentication Flow

    JWT is typically used in APIs for user authentication. The flow involves generating a JWT upon user login, and the client includes the token in subsequent requests to authenticate API calls.

    // Example of generating and verifying JWT in an Express route
app.post('/login', (req, res) => {
    const token = jwt.sign({ userId: '123' }, 'secretkey', { expiresIn: '1h' });
    res.json({ token });
});

app.get('/protected', (req, res) => {
    const token = req.headers['authorization'].split(' ')[1];
    jwt.verify(token, 'secretkey', (err, decoded) => {
        if (err) return res.status(403).json({ message: 'Forbidden' });
        res.json({ message: 'Protected content', userId: decoded.userId });
    });
});
  • JWT Verification

    To verify a JWT, the server checks the token sent by the client, decodes it, and ensures the signature matches the server's secret key.

    const token = req.headers['authorization'].split(' ')[1];
jwt.verify(token, 'secretkey', (err, decoded) => {
  if (err) return res.status(403).json({ message: 'Forbidden' });
  console.log(decoded); // Payload info like userId
});

Image Storage Learning Path

  • Image Storage Locally using Multer

    Multer is a middleware for handling image uploads in Node.js. It can store images locally on the server.

    const multer = require('multer');
const upload = multer({ dest: 'uploads/' });

app.post('/upload', upload.single('image'), (req, res) => {
    res.send('Image uploaded successfully');
});
  • Storing Images in Cloud (AWS S3)

    You can upload images to cloud storage like AWS S3 for scalability.

    const AWS = require('aws-sdk');
const s3 = new AWS.S3();

const uploadParams = {
    Bucket: 'your-bucket-name',
    Key: 'image.jpg',
    Body: fs.createReadStream('image.jpg'),
};

s3.upload(uploadParams, (err, data) => {
    if (err) console.log(err);
    else console.log('Uploaded to S3:', data.Location);
});
  • Serving Images from Cloud Storage

    After uploading images to cloud storage, you can generate a URL to serve the image to clients.

    const imageURL = data.Location; // URL to access the image
res.json({ imageUrl: imageURL });
  • Image Optimization

    Optimize images before uploading to save bandwidth and improve load times. Tools like `sharp` can help.

    const sharp = require('sharp');

sharp('image.jpg')
  .resize(800)
  .toFile('optimized-image.jpg', (err, info) => {
    if (err) console.log(err);
    console.log(info);
});
