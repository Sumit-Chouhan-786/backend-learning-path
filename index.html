Node.js, Express, MongoDB, and SQL Learning Path with Examples

A structured guide with examples to master full-stack development using Node.js, Express.js, MongoDB, and SQL.

Node.js Learning Path

  • Introduction to Node.js

    Node.js is a runtime environment for executing JavaScript outside of a web browser. It allows developers to use JavaScript for both front-end and back-end development.

    console.log("Hello, Node.js!");
  • Node.js Modules

    Node.js uses modules to organize code into reusable units. Modules like `fs` (File System), `http`, and `path` provide built-in functionality.

    const fs = require('fs');
fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});
  • Asynchronous Programming

    Node.js is built on asynchronous programming, meaning that it doesn't block other operations while waiting for some operations to complete.

    fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});
console.log('This line executes first!');
  • HTTP Module

    Node.js has an HTTP module to create HTTP servers and handle requests.

    const http = require('http');

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello, World!');
});

server.listen(3000, () => {
  console.log('Server running at http://127.0.0.1:3000/');
});
  • Event Handling in Node.js

    Node.js uses an event-driven model. The `EventEmitter` class allows you to create custom events.

    const EventEmitter = require('events');
const myEmitter = new EventEmitter();

myEmitter.on('event', () => {
  console.log('An event occurred!');
});

myEmitter.emit('event');

Express.js Learning Path

  • Setting up Express

    Express is a web framework for Node.js, simplifying the creation of servers and routes.

    const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello, Express!');
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
  • Routing in Express

    Express allows you to define routes for handling different HTTP methods like GET, POST, PUT, DELETE.

    app.get('/about', (req, res) => {
  res.send('About Us');
});

app.post('/contact', (req, res) => {
  res.send('Contact Form Submitted');
});
  • Middleware in Express

    Middleware functions are used to perform actions on the request before reaching the route handler.

    app.use((req, res, next) => {
  console.log('Request received');
  next();
});
  • Template Engines in Express

    Express can use template engines like EJS to render dynamic HTML pages.

    app.set('view engine', 'ejs');
app.get('/', (req, res) => {
  res.render('index', { title: 'Express' });
});
  • RESTful APIs

    Express is great for building RESTful APIs, allowing you to handle different HTTP requests for resources.

    app.get('/api/users', (req, res) => {
  res.json([{ name: 'Alice' }, { name: 'Bob' }]);
});

app.post('/api/users', (req, res) => {
  res.json({ message: 'User added' });
});

MongoDB Learning Path

  • Introduction to MongoDB

    MongoDB is a NoSQL database, meaning it stores data in a flexible, JSON-like format, making it great for handling complex data structures.

    const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost/myapp', { useNewUrlParser: true, useUnifiedTopology: true });
  • CRUD Operations in MongoDB

    CRUD (Create, Read, Update, Delete) operations are the basic functions to interact with MongoDB databases.

    const User = mongoose.model('User', { name: String });

const user = new User({ name: 'Alice' });
user.save().then(() => console.log('User saved!'));

User.find().then(users => console.log(users));
  • Aggregation Framework

    MongoDB's aggregation framework allows for complex data manipulations and transformations.

    User.aggregate([
  { $match: { name: 'Alice' } },
  { $group: { _id: '$name', count: { $sum: 1 } } }
]).then(result => console.log(result));
  • Schema and Mongoose

    Mongoose provides a higher-level abstraction to interact with MongoDB and define schemas for the data.

    const userSchema = new mongoose.Schema({
  name: { type: String, required: true }
});

const User = mongoose.model('User', userSchema);

const newUser = new User({ name: 'Bob' });
newUser.save().then(() => console.log('User saved!'));

SQL Learning Path

  • Introduction to SQL

    SQL (Structured Query Language) is used to interact with relational databases. You can define, manipulate, and query data stored in relational tables.

    -- Creating a table in SQL
CREATE TABLE users (
  id INT PRIMARY KEY,
  name VARCHAR(100)
);
  • Basic SQL Queries

    SQL queries are used to interact with the database. Common queries include SELECT, INSERT, UPDATE, and DELETE.

    -- Inserting data
INSERT INTO users (id, name) VALUES (1, 'Alice');

-- Selecting data
SELECT * FROM users;
  • Joins in SQL

    SQL joins allow you to combine data from multiple tables based on related columns.

    -- Inner join example
SELECT users.name, orders.amount
FROM users
INNER JOIN orders ON users.id = orders.user_id;
  • SQL Aggregate Functions

    SQL provides aggregate functions like COUNT, SUM, AVG, MIN, and MAX to perform operations on data.

    -- Count users
SELECT COUNT(*) FROM users;

-- Average order amount
SELECT AVG(amount) FROM orders;
